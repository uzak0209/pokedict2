# ポケモン構築ビルダー (Pokemon Team Builder)

実際の構築プロセスに基づいた、パーティ構築支援Webアプリケーションです。

## 🎯 構築理論

### 軸4体 + 補完2体

実際のトップランカーの構築プロセスに基づいています：

1. **行動保証のあるポケモン**から組み始める
   - チョッキオーガのように、テラス込みでほぼ動けるポケモン
   - 一撃、怯み以外のほぼ全てに対応できる

2. **相性の良いポケモン3体**を追加して軸を作る
   - 軸ポケモン合計4体で構築の核を形成

3. **重いポケモンを確認**
   - 軸4体で対応しきれないポケモンをリストアップ

4. **補完ポケモン2体**を追加
   - 重いポケモンへの対策枠
   - 受けループ対策の積みポケモン
   - クッション役など

### Top30対策表

使用率Top30のポケモンに対する対策状況を可視化：
- タイプ相性、素早さから有利不利を算出
- ユーザーが修正可能
- 修正情報を蓄積して精度向上

## 技術スタック

- **Svelte** + **TypeScript** (軽量・高速)
- **Vite** (ビルドツール)
- **TailwindCSS v4** (スタイリング)
- **Clean Architecture** (バックエンド - Rust)

## 開発

### 前提条件
- Node.js 18以上
- npm または yarn

### インストール

```bash
cd frontend
npm install
```

### 開発サーバーの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開きます。

### ビルド

```bash
npm run build
```

## プロジェクト構造

```
frontend/
├── src/
│   ├── lib/              # Svelteコンポーネント
│   │   ├── TeamBuilder.svelte    # メインコンポーネント
│   │   ├── PokemonSlot.svelte    # ポケモンスロット
│   │   └── TypeBadge.svelte      # タイプバッジ
│   ├── types/            # TypeScript型定義
│   │   └── pokemon.ts    # Pokemon関連の型
│   ├── data/             # 定数データ
│   │   └── typeColors.ts # タイプカラー定義
│   ├── stores/           # Svelte stores (状態管理)
│   ├── App.svelte        # アプリケーションルート
│   ├── main.ts           # エントリーポイント
│   └── app.css           # グローバルスタイル
├── public/               # 静的ファイル
└── package.json
```
